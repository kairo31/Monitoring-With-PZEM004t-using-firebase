Simulasi Alur Sistem (Langkah Demi Langkah)

Ikuti data dari sensor sampai ke rekomendasi AI

Tekan "Langkah Berikutnya" untuk melihat apa yang terjadi di setiap tahap. Setiap siklus menggunakan data sensor acak yang baru.

Progres simulasi satu siklus Langkah 0 dari 6
1
🔌 ESP32 membaca sensor
ESP32 membaca PZEM004t (tegangan, arus, daya, energi) via Serial2 dan DHT22 (suhu) via pin 4. Loop ini berjalan terus-menerus setiap 2 detik.
Tegangan: — VArus: — ADaya: — WSuhu: — °CkWh: —
2
🔥 Push ke Firebase Realtime Database
ESP32 menulis data ke /Monitoring setiap 2 detik (dashboard live) dan menambahkan record ke /history setiap 60 detik sebagai data latih AI.
/Monitoring ← data live/history ← tiap 60 detik
3
⚙️ GitHub Actions cron aktif
Setiap 15 menit, GitHub Actions otomatis berjalan. Ia mengunduh model AI dari Google Drive (atau cache) lalu menjalankan skrip rekap harian dan inferensi AI.
Cron: */15 * * * *sarimax_bundle.pklmodel_rl_budget.zip
4
📈 SARIMAX meramalkan daya
Model SARIMAX membaca data historis + status perangkat sebagai variabel eksogen, lalu memprediksi 6 langkah ke depan. Hasil mentah disanitasi: nilai negatif dibuang, outlier diperhalus dengan blending 70% daya sekarang + 30% prediksi.
Forecast mentah: — WSetelah sanitasi: — W6-step: —
5
🤖 Agen RL (PPO) memilih aksi
Agen PPO menerima state 15-dimensi: daya aktual, suhu, flag perangkat aktif (AC/TV/dll), jam, hari, status peak hour, progres bulan, gap budget, dan forecast SARIMAX. Output: satu dari 5 aksi penghematan.
Dimensi state: 15Aksi: —Gap budget: — Rp
6
📊 Hasil → Firebase → Dashboard
Semua output AI ditulis ke /Hasil_AI. Dashboard membacanya secara realtime dan menampilkan prediksi watt, rekomendasi RL, proyeksi biaya bulanan, dan badge penghematan.
Prediksi biaya harian: — RpStatus: —Badge: —
Kalkulator Prediksi Biaya Perangkat

Simulasi Komposisi Perangkat → Prediksi Konsumsi & Biaya

Masukkan perkiraan persentase waktu aktif tiap perangkat dalam sebulan (tidak harus total 100% — sistem otomatis normalisasi). Ini mensimulasikan output SARIMAX-like dari sistem.

Simulator ini tidak terhubung ke sensor atau Firebase — seluruh data adalah hasil perhitungan lokal di browser Anda.
